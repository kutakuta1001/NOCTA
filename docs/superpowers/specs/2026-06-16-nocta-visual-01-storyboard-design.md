# 設計書: NOCTA Visual 01 — HTMLアニメーション・ストーリーボード

作成日: 2026-06-16
ステータス: 承認済み（CEO 2026-06-16）

## 背景・目的

NOCTA Visual 01 はブランドコンセプト映像（藤の花に佇む少女・3シーン・3:4縦・約25秒）。
当初 Runway / Kling 3.0 Pro で AI 動画生成を試みたが品質が不足し停止。
今回は方針転換し、**HTML製のアニメーション・ストーリーボードをスライド形式で流す**ことで、映像の雰囲気と演出意図を他者に伝える手段とする。NOCTA の「Code」領域の強みを活かし、品質を完全にコントロールする。

正本（コンテンツ）: 既存絵コンテ `drafts/pv_storyboard.md`。キー画像は1枚のみ（藤の少女）。

## 成果物

- `website/pv/index.html` — 単一の self-contained HTML（素の CSS/JS・外部依存はフォント CDN のみ）
- `website/pv/` 配下に web 最適化したキー画像
- 準公開URL: `https://kutakuta1001.github.io/NOCTA/pv/`（ナビ未リンク・直URL共有。brandkit.html と同じ準公開方式）

## 構成（5パート）

| # | パート | 尺 | 演出 |
|---|---|---|---|
| 0 | タイトルカード | ~3s | warm-black にシルバーで「NOCTA Visual 01」+ サブタイトル |
| 1 | Scene 01 寄り | ~9s | 画像 scale 1.0→1.15 ズームイン・花びら降下・キャプション「寄り / 静寂の中で出会う」 |
| 2 | Scene 02 クローズ | ~6s | 顔付近へズーム+右ドリフト・前景に花びら・キャプション「クローズ / 内省」 |
| 3 | Scene 03 引き | ~10s | ズームアウト・花びら大量・画面が白へフェード・キャプション「引き / 世界に溶けていく」 |
| 4 | エンドカード | ~3s | 白→warm-black・`NOCTA.` ロゴ（シルバー）+ tagline |

各シーンの caption / duration / 画像 transform は **JS 設定配列**で管理し、調整を容易にする。

## レイアウト

- 3:4 縦ステージを warm-black 背景に中央レターボックス配置。レスポンシブ（ビューポートに収まるよう `min()` で拡縮）。

## 再生・操作

- 自動進行（各パートの duration を JS タイマーで遷移）。最後はエンドカードで停止。
- コントロール: 再生/一時停止トグル・←→キー&画面クリックでパート送り戻し・下部にパートドット（進捗表示・クリックでジャンプ）・リスタート。
- `prefers-reduced-motion: reduce` 時はアニメーション（ズーム・花びら）を抑制し、静止＋キャプションのみで進行。

## ブランド整合（紫と銀の役割分担）

- **chrome**（タイトル/エンド/キャプション/操作UI）= シルバー `#B8B4AE`・warm-black `#0A0906`・オフホワイト `#E8E0D0`・EB Garamond / Syne。
- **コンテンツ**（藤＝紫/ラベンダー）= 作品世界としてそのまま保持。
- これにより「現行ブランド=単一シルバー」と「作品=紫」が衝突せず両立する（額装＝銀、絵=紫の関係）。

## 画像の扱い

- 元画像 `outputs/pv/images/key/pixai-...zora girl #3.png`（1.6MB PNG）は outputs に温存。
- web 最適化版（長辺 ~1080px・150〜300KB の JPG）を `sips` で生成し `website/pv/` に配置・コミット。
- 根拠: 公開ページの表示に必要な軽量素材（`website/img/cat/*.png` のコミット前例に倣う）。Visual works の IPFS 参照とは別運用（ストーリーボード専用素材のため同梱が簡潔）。

## 音声

- 初版は無音（ブラウザの自動再生制限のため）。
- 将来 BGM を足せるよう `<audio>` 要素とミュートトグルのフックのみ用意（既定オフ・YAGNI）。

## 非対象（YAGNI）

- 複数キー画像の生成・差し替え（1枚運用）。
- 本格的な音声同期。
- 動画ファイル書き出し（pv_edit.py は今回不使用）。

## 検証

- ブラウザ表示確認（CEO）。
- HTML 構造チェック（タグ閉じ・参照画像の存在）。
- reduced-motion 動作確認。
