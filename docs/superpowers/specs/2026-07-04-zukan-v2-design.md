## 3. データ設計

### gems-data.js 追加フィールド
```js
kotobaJa: "情熱・勇気・威厳",   // 石言葉（伝承ベースで自作の表現。他サイトからのコピーはしない）
kotobaEn: "Passion, courage, dignity",
birthMonths: [7],               // 誕生石の月。非該当は []
```
誕生石マッピングは日本の2021年改訂リストに準拠（例: 3月アイオライト・8月スピネル・9月クンツァイト・12月タンザナイト/ラピスラズリ）。

### 新規21種
サファイア(青)・パール・ターコイズ・ラピスラズリ・シトリン・ローズクォーツ・翡翠・スピネル・モルガナイト・クンツァイト・ロードクロサイト・マラカイト・アンモライト・フローライト・ウォーターメロントルマリン・サンストーン・アイオライト・ラリマー・琥珀・ベニトアイト・オブシディアン

### flora-data.js（24種）
桜・梅・椿・チューリップ・すずらん・藤・バラ・紫陽花・花菖蒲・蓮・ひまわり・朝顔・桔梗・コスモス・彼岸花・金木犀・菊・山茶花・水仙・シクラメン・牡丹・ラベンダー・芍薬・ユリ(カサブランカ)

### 画像ライセンス
- 全件 Wikimedia Commons（CC0/CC BY/CC BY-SA/PDのみ）。並列エージェントがファイルページをWebFetchで実在・ライセンス・著者・直URLを検証
- 実装後に全imgUrlをHTTPで疎通確認（curl）
- クレジットはv1と同じくカード内の出典行に表示

## 4. 主要ロジック

### 今日の一石/一花（決定的・タイムゾーンはローカル）
```js
function todayIndex(total) {
  var d = new Date();
  var seed = d.getFullYear() * 10000 + (d.getMonth() + 1) * 100 + d.getDate();
  var h = seed % 2147483647;
  h = (h * 48271) % 2147483647;  // minstd一回転で日付の偏りをほぐす
  return h % total;
}
```

### サムネイル派生URL
```js
// https://upload.wikimedia.org/wikipedia/commons/a/bc/Name.jpg
// → https://upload.wikimedia.org/wikipedia/commons/thumb/a/bc/Name.jpg/1280px-Name.jpg
```
imgのonerrorで原本URLへフォールバック（原本より大きいサムネ要求などの失敗に備える）。

### 目次オーバーレイ
- fixedボタン（右下・railと干渉しない位置）→ position:fixedの全画面レイヤー
- 月チップ（1〜12月＋すべて）はフィルタとして機能。グリッドは色見本＋和名＋英名
- role="dialog"・ESCで閉じる・開いている間はbodyスクロールロック

## 5. エラーハンドリング
- 画像404: thumbnail→original→（両方失敗時）accentColorの単色プレースホルダー

- reduced-motion: v1の対応を全新機能にも適用（スムーススクロール・アニメ無効）
