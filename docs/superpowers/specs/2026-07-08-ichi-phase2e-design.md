# Ichi Phase 2e「草花の標本 — 自生と標本化」設計

作成: 2026-07-08 / モデル: Opus 4.8 / 実装SA: Sonnet・レビュー: Codex（CEO指定）
承認: CEO（両方＝自生＋標本化 / 植物4種すべて / 押し花の地は暖かい生成り beige）
状態: 実装計画へ

## 目的（CEOの依頼）

参考画像4枚（BLOOM MAISON の押し花フラットレイ／植物標本シート／吊るしたレースフラワーの草／押し花の額装作品）が示す
「花＋豊かな緑（草・つる・シダ・種穂）が、自然な余白をもって標本のように組み上がる」世界を、
今の Ichi「咲かせる」のコンセプトを変えずに実現する。CEOの3つの要望:

1. 花が散る・風で散る（既存）に加え、**草が生える**ようにする。
2. 参考画像のような花や草（草・つる・シダ・レースフラワー/種穂・小花の群れ）にする。
3. **最終的にこのようなものが自動で出来上がってくる** — 放置でも庭が自生し（両方選択）、押すと植物標本ボードに自動構成される。

憲法（継承・不変）: 引き算が上質・遅さの報酬・一回性・ゴールなし・無音でも成立。
CEO判断: 自動構成=両方（自生＋標本化）／植物=草・つる・シダ・レースフラワー/種穂・小花の群れ すべて／押し花の地=暖かい生成り(beige)。

## 設計要点

### 1. 草花4種（新しい生長物 kind）

花（既存7形・kind 実質 'flower'）に加え、庭の要素に `kind` を持たせ、緑の生長物を4種追加する。
すべて手続き生成・Canvas 2D・既存 petal()/mixRgb()/rgba() を流用。色は葉の基調セージ緑（#6E7A55）を
ベースに季節でわずかに寄せる（既存 palette と調和）。bloom（0→1）で開く点は花と同じ。

- **sprig（草・つる）**: 中心から伸びる緩い曲線の茎（2次ベジェ相当を線分近似）に、小さな楕円の葉を左右交互に連ねる。
  参考1/2 のタイム風。bloom で茎長・葉数が伸びる。rot/sway に追従。
- **fern（シダ）**: 直線〜緩曲の中軸（rachis）に、対になった羽状小葉（pinna）を先端へ向かって小さくしながら並べる。参考2。
- **umbel（レースフラワー／種穂）**: 一点から細い茎が放射状に伸び、先端に微小な点/小花。参考3。風で特に揺れやすい（軽い）。
- **floret（小花の群れ）**: 小さな5弁花を数輪（3〜6）、小さな範囲に点々と。参考1の小さな塊/かすみ草。既存の花描画を微小スケールで流用。

描画は drawFlower と同じ「translate→rotate→sway」座標系の中で行い、風・sway・押し花取り込みに自動追従する。
kind ごとに専用の描画関数（drawSprig/drawFern/drawUmbel/drawFloret）を用意し、drawEntity(ctx,o) が o.kind で分岐する
（既存 drawFlower は 'flower' 経路として維持し、既存挙動は完全不変）。

### 2. 生える（2経路）

- **なぞり中（player-driven）**: 現状の花に加え、軌跡近傍に草類が混ざって生える。速さで出し分け:
  速い（distance sampling が粗い）→ sprig/umbel など軽い緑、遅い→ 花・fern・floret。
  比率は「花が主役・緑は添え」を保つ（例: 生成の 30〜40% が緑）。既存の距離サンプリング spawn 経路に kind 選択を差し込む。
- **自生（ambient auto-growth）**: 18〜36秒に一度、庭のランダム位置に緑または小花が**ゆっくり**芽吹く（花ではなく緑・小花を優先）。
  ただし **「休息の充実度」**（restingCount、例: 緑を含む要素が一定数 N=28 前後）に達したら自生は止まり、庭が落ち着く。
  press で平面化・保存後にクリアされたり、散り/加齢で要素が減れば restingCount を下回り、また少しずつ芽吹く。
  これにより「無限増殖しない（引き算）」かつ「最終的に自然な庭が出来上がる（自動構成）」を両立する。
  自生は wind と同じくスケジューラ（nextSproutT）で管理し、reduce/farewelling 時は発火しない。

### 3. 散る・風（既存を緑に拡張）

- 花の散り（shed・落下パーティクル）・風（16-34秒・sway・onWind音）は既存のまま維持。
- 草・umbel も風でそよぐ（sway 係数は kind で軽重を調整。umbel は軽く大きく揺れる）。
- umbel/種穂からは「種」が舞う（既存の花びらパーティクルの緑版・色を緑/生成りに）。実装は既存 petals パーティクルに color を持たせて共用。
- reduce時: 風・散り・種飛びなし（静的に緑は付く）。

### 4. 押し花＝標本ボードに自動構成（最大の変更）

現状の press() は庭を現状のまま和紙に平面化。Phase 2e では **標本ボードに自動構成**する:

- 地: **暖かい生成り（beige, 例 #E4D9C4〜#D8C3A6 のグラデ/紙質）**。washi ではなく標本紙・厚紙の質感。
- 構成: 庭の要素（花・sprig・fern・umbel・floret）から代表を選抜し、**重ならないよう余白をあけて配置**する。
  - 大きな花を数輪、周囲に小花・緑・つるを散らす（参考1/2 の疎な標本レイアウト）。
  - 要素間の"つなぎ"に sprig（つる）を曲線で配置。
  - 各要素は「押された」表現（平面化＝影を消す・退色＝彩度と明度をわずかに落とす・輪郭をやや締める）で描く。
  - 配置は決定的乱数（seed）で再現可能に。要素数が少ない庭でも成立するようフォールバック（最低限の配置）。
- ラベル: 既存の漢数字日付を活かし、標本紙の隅に控えめな見出し（seed時は花名＋日付「薔薇 の 標本 · 七月八日」相当・汎用時は季節＋日付）。
  参考1の "BLOOM / MAISON" のような大見出しは NOCTA トーンに合わせ抑制（過度な装飾はしない・引き算）。
- 出力は既存同様 PNG（drawImage で canvas を平面化する経路を、標本レイアウト描画に置き換え）。透明前提は維持。

### 5. コンセプト保全・reduce・パフォーマンス

- 自生は遅く・まばら・充実で停止 → 瞑想性（引き算・遅さ）と能動性（自分で咲かせる）を守る。
- 無音でも成立（音は既存トグルのみ。緑の生長に新規音は足さない。将来 onWind 同様のフックは可）。
- reduce-motion: 自生・風・散り・種飛びなし（静的に緑は付き、押し花標本化は静止でも機能）。
- 総数上限（既存 MAX_FLOWERS=400）を全 kind 合算で踏襲。自生の restingCount 停止で常時rAFの暴走を防ぐ
  （needLoop は要素/パーティクルが動いている間のみ true・自生の待機は setTimeout で rAF を起こさない）。

## 段階（計画の分割）

規模が大きいため実装計画を2段に分ける（各段が独立してテスト可能・公開可能）:

- **2e-1「草花4種と生える・散る・風」**: kind 導入・drawSprig/Fern/Umbel/Floret・なぞりミックス・自生（restingCount停止）・
  風/sway/種パーティクルの緑対応。press は当面「現状の庭をそのまま平面化」で緑も写る（既存経路で自動反映）。
- **2e-2「押し花＝標本ボード自動構成・beige地」**: press() を標本レイアウトに置換・beige地・退色・つなぎのつる・ラベル。

## リスク

| リスク | 対応 |
|---|---|
| 自生で庭が騒がしくなり瞑想性を損なう | 遅い間隔・restingCount で停止・緑優先。目視で調整可な数値に集約 |
| 常時rAFでバッテリー/perf悪化 | 自生待機は setTimeout（rAF起こさない）・充実で停止・総数上限踏襲 |
| 草4種が「本物感」に届かず記号的になる | petal流用＋葉脈/羽状/放射の手続き差・季節色。スクショ目視で調整 |
| 標本ボードの自動配置が窮屈/重なる | 余白マージン・要素選抜・決定的乱数・少数フォールバック。目視調整 |
| beige地が既存の暗い庭トーンと乖離 | press は元々別コンテキスト（和紙地）。beige は標本作品として独立トーンで可 |
| kind 追加で既存描画/press が退行 | drawFlower/'flower'経路は完全不変・回帰スイート全緑を必須ゲートに |
