# Ichi Phase 2g「自動で仕上がる草花の一枚」設計

作成: 2026-07-09 / モデル: Opus 4.8 / 実装SA: Sonnet・レビュー: Codex（CEO指定）
承認: CEO（自生を大きく画面いっぱいに・参考画像の押し花ボードのような美術的な一枚に勝手に仕上がる）
CEO判断: 密に敷き詰める／完成形まで育ち静かに落ち着く／トーンは参考画像の明るい生成り(beige)基調に寄せる
状態: 実装計画へ

## 目的（CEOの依頼）

好評だった「自動で入ってくる自生」と「タップ（＝突風クリア）での取り消し」を土台に、
**自生部分を大きく画面いっぱいに拡張し、放置すると参考画像（押し花の植物標本ボード "BLOOM MAISON" 系）のような
明るい生成り基調の"美術的な一枚"に勝手に密に仕上がっていく**ようにする。完成したら静かに落ち着く。

憲法（継承）: 一回性・ゴールなし・無音でも成立・reduce対応。「引き算」はこのフェーズでは
「密度」方向へCEOが明示的に舵を切ったため、"仕上がったら静止する（際限なく増やさない）" で担保する。

## 設計要点

### 1. 自生を「画面いっぱいの構成」まで拡張（アイドル駆動は維持）

- **目標充実度 FILL_TARGET**: 28 → **44**（密に敷き詰める・画面いっぱい）。FILL_TARGET 到達で自生停止＝「静かに落ち着く」。
- **サイズを大きく・変化をつける**: ambientSprout の要素サイズを拡大し変化を持たせる。
  - 主役の大輪級（r 70〜100）を自動配置に数輪、中サイズの花（r 40〜65）、小花・緑（r 26〜46）を混在。
  - 花:緑の比率は「花が主役・緑がつなぎ」を保つ（緑 ~40%）。fill が進むほど大小のリズムが出るよう配分。
- **配置**: 既存の「空いた最遠点」バイアス（候補点から既存要素・直近ポインタに最も遠い点）を維持しつつ、
  密度が上がっても大きな要素が過度に重ならないよう、候補生成時に大要素は間隔を優先（小要素・緑は隙間を埋める）。
- **ペース**: 既存のアイドル駆動（最後の操作から IDLE_MS=3000 後に開始・以後 SPROUT_INTERVAL 3〜6秒ごとに1つ）を維持。
  FILL_TARGET まで時間をかけて「一枚が仕上がっていく」。操作でリセット（既存）。
- reduce時は自生なし（静的）。

### 2. 背景の移ろい（暗い庭 → 明るい生成り）

- **fillRatio** = min(1, flowers.length / FILL_TARGET) を算出。ステージ背景を、現行の warm-black（暗い光の庭）から
  **明るい生成り beige（例 #E4D9C4〜#DED3BC）へ fillRatio で lerp**。空＝暗、完成＝明るい生成り。
  - これが「自動で明るい美術作品に仕上がっていく」体験の核。tap 追加も fillRatio に寄与し明るくなる／突風クリアで空＝暗へ戻る。
  - 実装: `.hana-stage` 背景（既存の季節連動の光だまり CSS）を、fillRatio に応じて JS から生成り側へ補間（CSS変数 or 直接 background 設定）。
    グレイン/光だまりは残しつつ、明るい地でも成立するトーンに。
- **明るい地での花・緑の視認性**: 花色（petals）は明地でも映える中間色主体なので基本は維持。ただし「暗い庭で光る」前提の
  芯グロー/淡い外周は明地で沈む/浮く恐れ→ fillRatio が高い（明るい）ときは花の描画をわずかに"押し花寄り"（彩度・
  コントラストを軽く上げ、グローを弱める）に補正し、参考画像の押し花トーンに寄せる。緑（GREEN）は明地で自然。
- reduce時は移ろいなし（初期トーン維持）。

### 3. 完成して落ち着く

- FILL_TARGET 到達で自生停止（既存 RESTING の考えを FILL_TARGET に拡張）。背景は明るい生成りで静止。
  微風（sway）は継続（呼吸するそよぎ）だが要素は増えない＝「一枚の作品として落ち着く」。
- 要素が減れば（突風クリア／将来の散り）また FILL_TARGET へ向けて仕上がりが再開。

### 4. 維持するもの

- タップで花追加（アイドルリセット・fillRatio 寄与）。突風クリア（全部散る→空→暗→再びアイドルで仕上がる）。
  なぞりミックス・風・長押し大輪・press（押し花標本ボードは独立して従来通り）・reduce・無音。
- 音は自生に付けない（従来通り無音で成立）。

## 変更ファイル

- `website/flora/hana.js`: FILL_TARGET(28→44)・ambientSprout のサイズ/配分/配置拡張・fillRatio 算出・背景の暗→生成り補間

## 検証設計（Playwright + Codex）

- 新規 `verify-hana-phase2g.js`:
  (a) FILL_TARGET拡張: `__fastGrow` で放置すると entityCount が 44 付近まで増え、そこで停止（44を大きく超えない）。
  (b) サイズ拡大: 自生要素に大輪級（r>=70）が含まれ、サイズに強弱がある（全部小さくない）。
  (c) 背景移ろい: fillRatio 上昇に伴い stage 背景の明度が上がる（空≒暗い→充実≒明るい生成り。背景色サンプルの輝度比較）。空/クリア直後は暗い。
  (d) 完成静止: FILL_TARGET到達後さらに時間を進めても 44 を超えない。
  (e) reduce: 自生・背景移ろいなし（暗いまま静的）。
  (f) 維持: タップ追加・突風クリアで空→暗へ・その後アイドルで再び増える。press は従来通り beige 標本ボードを返す。
  (g) エラー0。
- 既存 hana スイート（util/forms/seed/engine2/press/mount2/perflower/phase2b/season/oto/kaze/taika/hizuke/kyou/integration2/leaf/wind2/phase2e/phase2f）＋iro/fude/zukan 回帰維持。
  特に phase2e/phase2f の自生・背景前提テストは FILL_TARGET/背景変更で期待値更新の可能性→報告（__noSprout 隔離を活用）。
- コードレビュー: Codex（`codex-review.sh custom` でブランチ差分・2周・Critical必修正）。
- スクショ/実機目視（必須）: 放置で暗い庭→明るい生成りの密な草花の一枚に仕上がる過程・完成形が参考画像の質感に近いか（CEO最終採否）。

## リスク

| リスク | 対応 |
|---|---|
| 明るい地で花の"光る"表現が沈む/浮く | fillRatio高で花描画を押し花寄りに軽補正・グロー弱め。スクショ目視で調整 |
| 密度44＋大輪で30fps描画が重い | FILL_TARGET/サイズは数値で調整可・上限 MAX_FLOWERS(400) 内・完成で増加停止 |
| 暗→明の移ろいが唐突/濁る | fillRatioでなめらかlerp・グレイン/光だまり維持・目視で色を追い込む |
| 既存の暗い庭の世界観が薄れる懸念 | 操作中(空〜低fill)は暗いまま・明るくなるのは自動で仕上がった時だけ＝棲み分け |
| phase2e/2f テストの前提変更で退行 | __noSprout隔離・FILL_TARGET/背景の期待値は正当更新のみ・報告 |
| press標本ボードとライブ明るい構成の重複感 | press はPNG保存の keepsake として独立維持（役割が違う） |
