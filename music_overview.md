# NuWord 制作フロー概要

---

## フェーズ1: トレンド分析（Claude Code）

- チャート傾向・参考楽曲・差別化案を生成
- Sunoで仮試聴して方向性を確認
- 承認ゲート①: 方向性確定

---

## フェーズ2: 楽曲制作（CEOがメインで作る）

### Claude Codeがやること
- コード進行・ベースラインのMIDI叩き台を生成
- 歌詞草稿A/B/Cを生成（修正前提）
- music_spec.mdで設計方針を提示
- ※メロディMIDIは生成しない（CEOがオリジナルで作るため）

### CEOがやること（Studio One）
- Sunoの音源をLALAL.AIで分離してパーツを取り出す
- drums.wav・bass.wavをStudio Oneに読み込む
- chord_draft.midをStudio Oneに読み込んで確認
- メロディをオリジナルで作る（ここが最重要）
- 歌詞を選んで自分の言葉に修正する
- アレンジが固まったら「アレンジOKです」と報告

### CEOがやること（Synthesizer V）
- アレンジ確定後にSVPファイルを生成（/phase2-svp）
- ボーカルの感情・ピッチを微調整する

### 承認ゲートの変更
- ゲート②: 楽曲仕様書確認
- ゲート③: 歌詞選択
- ゲート④: Sunoパーツ＋MIDIで雰囲気が掴めたか
- ゲート⑤: アレンジ確定後にのみ実行（「アレンジOKです」が合言葉）

---

## フェーズ3: PV制作（Claude Code）

- コンセプト・絵コンテ・Midjourney/Runwayプロンプトを生成
- 承認ゲート⑥: PVコンセプト確認
- 承認ゲート⑦: 映像素材選択（CEOが実行・選択）

---

## フェーズ4: リリース準備（Claude Code × 4エージェント並列）

- SNSカレンダー・KPI・プレスリリース・著作権確認を並列生成
- 承認ゲート⑧: コンテンツ一括確認

---

## フェーズ5: リリース当日（Claude Code）

- go_live_checklist.md を生成
- 承認ゲート⑨: 最終GO判断（CEOのみ）

---

## Claude Codeの役割まとめ

| 役割 | Claude Code | CEO |
|---|---|---|
| トレンド分析 | 実行 | 承認 |
| コード進行MIDI | 叩き台生成 | 修正・採用判断 |
| メロディ | 作らない | オリジナルで作る |
| 歌詞草稿 | 叩き台生成 | 修正・確定 |
| SVP生成 | アレンジ確定後に実行 | Phoneme・感情調整 |
| ミックス・マスタリング | 関与しない | Studio Oneで実施 |
| PVプロンプト | 生成 | 実行・選択 |
| SNS投稿 | 下書き生成 | 手動投稿 |
