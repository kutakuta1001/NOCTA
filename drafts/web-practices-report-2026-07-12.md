## 適用しない項目と理由

- **CLAUDE.mdを200行以内に収める**: 2026-07-04のベストプラクティスレビューで既に検討済み・不採用（NOCTAは意図的に詳細なルール群を保持する方針。段階的開示はスキル分離で既に実践済み）。今回の発見（プロンプトキャッシュ効率化の観点からの言及）は同趣旨の再確認であり、判断を維持する。CLAUDE.mdはセッション中に内容が安定しているため、キャッシュヒット率への影響は限定的と判断。
- **Webhookによるイベント駆動型の完全自動化**（音源完成→ビジュアル自動生成、コードpush→レビュー自動開始等）: R-13「対話してから作業する」「CEOのGoサインなしにファイル生成を開始しない」という設計思想と直接衝突するため不採用。ただしwebsite/のGitHub Code Review App（PR作成時の自動レビュー）は同種の自動トリガーとして既に例外的に運用中（対話性を損なわない範囲のため）。
