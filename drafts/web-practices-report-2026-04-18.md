## モデル使い分け詳細（今回のフォーカス）

今回の目的（Opus 4.7 vs Sonnet 4.6 の使い分け検討）に対して、公式ドキュメントから得た仕様差と Web 知見を組み合わせると以下の判断基準が導出できます。

### 現在の R-09 課題
- Opus 4.7 には **拡張思考（Extended Thinking）がない**（Sonnet 4.6・Haiku 4.5 にはある）
- 「深い推論」が必要なタスクでは Sonnet 4.6 + 拡張思考が Opus 4.7 を上回る可能性がある

### 提案: タスク特性別モデル選択

| タスク特性 | 推奨モデル | 理由 |
|-----------|-----------|------|
| 繰り返し・定型・短い出力 | Haiku 4.5 | 最安・最速 |
| 創作・文章・SNS・仕様書 | Sonnet 4.6 | バランス良好 / 拡張思考も使える |
| 深い推論・ステップバイステップ分析 | Sonnet 4.6（拡張思考モード） | 拡張思考あり・Opusより安価 |
| SVP生成・最高品質実行・重要設計 | Opus 4.7 | 最高知能・適応的思考・128k出力 |
| エージェント型コーディング | Opus 4.7 | 公式が「大幅な改善」と明記 |

### コスト比較（参考）
- Haiku 4.5: $1/$5/MTok
- Sonnet 4.6: $3/$15/MTok
- Opus 4.7: $5/$25/MTok（Sonnetの約1.67倍）

### R-09 更新案（提案のみ・自動適用しない）
```
R-09 現行:
- Opus: SVPファイル生成・複雑なコード生成・重要な設計判断・品質が特に重要な判断

R-09 提案追記:
- Opus 4.7: SVP生成・エージェント型コーディング・重要設計判断・最高品質実行
- Sonnet 4.6（拡張思考）: 詳細な推論・深い分析が必要な場面（歌詞の深い検討・楽曲構成の論理的分析）
※ Opus 4.7 は拡張思考（Extended Thinking）なし・適応的思考（Adaptive Thinking）あり
```
