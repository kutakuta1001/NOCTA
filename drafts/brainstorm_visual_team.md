# ブレインストーミング: NOCTAビジュアル・映像チーム設計
日付: 2026-03-30

---

## 核心的な方向性

「楽曲PV制作の補助」ではなく、**ビジュアル作家・映像クリエイターとしてのNOCTA**を確立することが主目的。
静止画アートと映像作品を独立した創作活動として育て、将来的にNFT販売・HP展示へ展開する。

---

## 担当コンテンツの範囲

### 一次目的（クリエイターとして）
- **静止画アート**: AI生成・コラージュ・デジタルペインティング等の単体作品
- **映像作品**: ループ映像・シネマティック・実験映像等の単体作品

### 二次目的（楽曲との連動・ブランド）
- 楽曲PV（NuWord等、曲リリースと連動したビジュアル）
- ブランドイメージ・アイデンティティ視覚化
- SNS投稿用静止画・短尺動画
- HP Visual セクション・ブログ用ビジュアル

### 将来展開
- NFT販売（Zora等）
- HP Visualセクションへのポートフォリオ展示

## AIエージェントチームの役割分担

### AIがやること
| 役割 | 内容 |
|------|------|
| Midjourneyプロンプト生成 | スタイル・ムード・構図・カメラ設定を含む詳細プロンプト |
| Klingモーションプロンプト生成 | 動きの方向・速度・カメラワーク等を含むプロンプト |
| スタイルガイド管理 | シリーズ・シーズンの一貫性を保つパラメータの蓄積と参照 |
| コンセプト壁打ち（補助） | 世界観・色調・参照作品のブレインストーミング相手 |

### CEOがやること
- コンセプト・世界観の最終方向性を決める
- Midjourney / Kling を実際に実行する
- 生成結果の採用・不採用を決める
- CapCutでの最終編集
- 発表タイミング・プラットフォームの判断

## リポジトリ管理方針

**project_NOCTA に統合**（visual_NOCTA の分離はしない）

理由:
- HP の Visual セクションと同リポジトリで自然に連携できる
- song/ ブランチとの楽曲連動が容易
- 管理コスト（CLAUDE.md・Netlify・設定）を増やさない

### ディレクトリ構造（新設）

```
project_NOCTA/
└── visual/
    ├── style-guide.md        ← シリーズ共通のスタイル定義
    ├── standalone/           ← 単体作品ごとのフォルダ
    │   └── [作品名]/
    │       ├── concept.md    ← 世界観・コンセプトメモ
    │       └── prompts.md    ← Midjourney / Kling プロンプト集
    └── linked/               ← 楽曲連動PV・ブランドビジュアル
        └── NuWord/
            ├── concept.md
            └── prompts.md
```

実画像・動画ファイルはgitに入れない（Zora URL / YouTube URL で参照）
