## 0. 結論（先に要点）

チープさの正体は「機能の不足」ではなく**質感と時間の設計の不足**。したがって提案の中心は
機能追加ではなく、(A) 描画の脱ベタ塗り、(B) 時間の流れを瞑想のテンポに落とすこと、の2つ。
リサーチの最大の発見は「**上質さは足すことではなく引くこと・制約から生まれる**」
（Awwwards受賞サイトは2色パレット、生け花サイトはモノクロームのみ、瞑想アプリの設計原則は
"restraint is care"）。Ichi は既に「1画面1種・静かに眺める」という正しい骨格を持っている。
その骨格に質感と時間を与えることが Phase 2 の仕事になる。

## 1. 診断 — チープに見える5つの原因

現状スクリーンショットとコードから特定した具体因。いずれもリサーチの知見と一対一で対応する。

| # | 原因 | 現状 | リサーチの裏付け |
|---|------|------|-----------------|
| 1 | 花びらが単色ベタ塗り | 1花=花びら1色+花芯1色のフラット塗り。濃淡・透過・重なりの階調なし | 「半透明の重なり・光の透過感がデジタルの平坦さを消す」。本物の花びらは光を透かす |
| 2 | 背景が「素の黒」 | #0A0906の完全ベタ。光だまり・霞・空気感なし | Gyuで実証済み（ビロード風光だまりで実在感が出た）。光源と奥行きは高級感の最短経路 |
| 3 | 全ての花が同一平面 | 重なり順だけで、ボケ・遠近・スケールの空気遠近なし。「シールを貼った」ように見える | 被写界深度・ボケは平面イラストとの決定的な差 |
| 4 | モーションが機能的 | scale+回転の開花のみ。咲いた後は完全静止。風・揺らぎ・舞いがない | 「物理法則に従う動きは作り物っぽさを消す」（Verlet物理で6万花びらの事例あり）。teamLabは「常に生成中」であることが本物感の核 |
| 5 | 儀式の不在 | 全画面に入る/出る瞬間の演出なし。押し花の瞬間も軽い | 瞑想体験は「開始と終了の儀式」（ろうそくの点灯/消灯に相当）で日常と区切られる |

補足: 図鑑側（写真・1画面1種・ゆっくりズーム）は方向として正しく、大きな問題はない。
チープさの主因は「咲かせる」遊びレイヤーの描画品質と、体験全体の時間設計にある。

## 2. 北極星 — このアプリは何か

**「花の美しさと多様さを思い出させ、瞑想の起点になる装置」**（CEOの言葉から）

リサーチから導いた設計原則（Ichi 憲法として Phase 2 全体に適用）:

1. **引き算が上質さ**: 色数を絞る。UIを減らす。密度を上げない。「何を削るかが最大の設計判断」
2. **遅さを報酬にする**: 速い操作を罰するのではなく、ゆっくりした操作にこそ美しい反応を返す
   （Pause の実証知見: 遅い操作の報酬化が精神的作業負荷を下げる）
3. **二度と同じ画にしない**: 開花の速度・角度・色に揺らぎを注入。「常に生成中」が本物感と
   無常観の両方を作る（teamLab / Generative.fm の核）
4. **ゴールをつくらない**: スコア・達成率・コレクション率は瞑想性と逆方向。押し花保存は
   「成果」ではなく「余韻」として残す
5. **無音でも成立させ、音は最小の生成音から**: Pause / thisissand / 禅ガーデン系は無音で成立。
   足すなら「なぞった瞬間の短い減衰音」（Web Audio生成・素材ファイル不要）まで

## 3. 改善提案 — 3層構造・優先度付き

### A層: 質感の底上げ（「チープ」の直接解消）— 最優先

| 提案 | 内容 | コスト |
|---|---|---|
| A-1 花びらの脱ベタ塗り | 花びらごとに radial/linear グラデーション（付け根濃→先端淡）+ globalAlpha の重なりで光の透過感。花びら1枚ずつ僅かに色相・明度を揺らす | 小 |
| A-2 背景に空気を | 素の黒 → 夜の庭の空気（下方に淡い光だまり・上方に微かな青みのグラデ・Gyuのビロード知見流用）。季節パレットに応じて背景の色温度も変える | 小 |
| A-3 奥行きの導入 | 花に手前/奥の2〜3層を持たせ、奥の花は小さく・淡く・僅かにぼかす（shadowBlur流用）。空気遠近で「庭」になる | 中 |
| A-4 グレイン | 画面全体に低opacity（2〜5%）のノイズ。既にHP本体で使っている `.grain` の流儀。気づかれない程度が要点 | 小 |
| A-5 開花後の微風 | 咲いた花が完全静止しない。数十秒周期の極小の揺れ（回転±1〜2°・scale±1%）。「生きている」感の最小実装 | 小〜中 |

### B層: 瞑想メカニクス（「瞑想の起点」の実装）— A層と同時期

| 提案 | 内容 | コスト |
|---|---|---|
| B-1 遅さの報酬化 | 現行「速い=小さい花」を徹底: 速くなぞると淡く小さくまばらに、**ゆっくりなぞるほど大きく豊かに深い色で咲く**。ユーザーの手が自然に減速する（Pause型） | 小〜中 |
| B-2 呼吸のテンポ | 開花アニメを現行700ms→呼吸に近い時間構造（約4秒で開き、静止し、揺れに移る）。急がない時間をアプリ側が体現する | 小 |
| B-3 入場/退場の儀式 | 入場: 暗闇から一輪がゆっくり浮かび上がるフェードイン（2〜3秒）。退場: 閉じる時に咲いた花々が静かに閉じてから暗転。「ここは特別な時間」の区切り | 小〜中 |
| B-4 無常（花は散る） | 咲いてから長時間経った花は、時折花びらを1枚ずつ手放して淡く散る（パーティクル）。庭は永遠に固定されず、緩やかに移ろう。押し花は「移ろうものを留める」行為として意味が立つ | 中 |
| B-5 なぞり音（任意・後段） | なぞった瞬間だけ鈴/水滴的な短い減衰音を Web Audio で生成（素材不要・ライセンスリスクゼロ・自動再生制限に配慮しデフォルトはオフ、控えめなオンUI） | 小 |

### C層: 図鑑側の磨き（「花の美しさ・多様さを思い出させる」）— 後段でも可

| 提案 | 内容 | コスト |
|---|---|---|
| C-1 間の再配分 | 各花のセクションでテキスト/UIの情報密度をさらに下げ、花の周囲の余白（空気）を増やす。生け花の思想=一本の花+周りの空気に意味 | 小 |
| C-2 今日と連動する景色 | 実際の日付から季節を判定し、「咲かせる」の初期パレットと図鑑の背景の色温度を自動で合わせる（七十二候アプリの一回性）。「今日開いた人だけの景色」 | 小 |
| C-3 図鑑→咲かせるの橋 | 眺めていた花のaccentColorを持って「この花の色で咲かせる」導線（将来の24種連動の最小版）。図鑑と遊びが分断されない | 中 |

### あえてやらないこと（引き算の明文化）

- **ゲーミフィケーション全般**（達成率・スタンプ・連続日数）: 瞑想性と逆方向
- **環境音BGMの初期導入**: 無音で成立させるのが先。導入するとしてもB-5の操作音まで
- **一輪挿し・絵本への派生**: CEOの意向どおり優先度を下げ、バックログに保管
  （B-4「花は散る」が入ると、「留めたい」動機が生まれ、将来の一輪挿しの意味も強くなる）
- **WebGL化・Kuwaharaフィルター等の大規模描画改修**: 差別化効果は最大だがコスト大。
  A層(Canvas 2Dのまま)の効果を見てから判断

### 6-3. 横断的な発見（要点の再掲）

1. **制約が上質さを作る**: 2色パレット・モノクローム・機能の少なさ。「何を削るかが最大の設計判断」
2. **遅さ・静止を報酬にする**: Pause（実証研究）と teamLab（止まるほど咲く）が同じ結論に到達している
3. **一回性が本物感を作る**: teamLab「常に生成中」・Generative.fm「二度と同じ演奏がない」・七十二候「今日だけの景色」
4. **無音でも瞑想は成立する**: Pause・thisissand・禅ガーデン系は無音。音は「説得の一層」であり必須ではない
5. **儀式が日常と区切る**: 入場のフェード・数呼吸の強制・60秒の消滅・終末フェーズ。開始と終了の設計が体験の格を決める
6. **グレイン・質感は「気づかれない程度」（2〜5%）**: 過剰にすると逆に安っぽくなる

### 6-4. 技術メモ（実装フェーズへの引き継ぎ）

- 現行 hana.js は Canvas 2D・パラメトリック手続き花（drawFlower）。A層は全て延長で実装可能
- Verlet物理の揺れは Canvas 2D では少数花に留める（WebGL移行しない限り大量は不可・piellardj実測）
- Kuwahara水彩化は WebGL/GLSL 必須（フラグメントシェーダ）。効果最大・コスト大の将来候補
- 音は Web Audio API の生成音（sine/triangle減衰）なら素材ファイル不要・ライセンスリスクゼロ。
  ブラウザ自動再生制限により初回ユーザー操作後のみ再生可・デフォルトオフ+分かりやすいオンUIが必須
