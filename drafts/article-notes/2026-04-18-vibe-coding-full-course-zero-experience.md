## 要約（3〜5行）
コーディング未経験者がVibe Coding（自然言語でAIに指示してアプリを構築する手法）でリアルなアプリを作るための完全ガイド。Claude + Claude Code / Cursor / Replit などのツール選択から5つの実践ルール、プロジェクト段階的ロードマップまでを網羅。「あなたはアーキテクト、AIはビルダー」という役割分担が核心。

## 主なポイント
- ツール選択: Claude + Claude Code（推奨）/ Cursor（IDE型）/ Replit・Bolt（ブラウザ型）
- 5ルール: ①プロンプト前に考える ②一度に一つだけ ③スクリーンショットを使う ④ノートを残す ⑤動く版を保存する
- `notes.md` に設計判断・アーキテクチャ・既知バグを常時保存 → コンテキスト引き継ぎが可能に
- 6プロジェクト段階: 個人ダッシュボード → 他者向けツール → DB付き → 認証付き → AI統合 → 販売可能なもの
- 「ツールホッピングは最大の時間無駄」— まず1つで3プロジェクト完成させる

## NOCTAへの関連メモ
「notes.md に設計判断を常時保存」はNOCTAの handoff.md（R-06）・CLAUDE.md 設計原則と完全に一致。R-14（セッション再開できる状態を保つ）の裏付けになる。また「一度に一つのタスク」はNOCTAのフェーズ別承認ゲート設計（①〜⑨）と同じ思想で、設計の正しさを外部から検証する参考になる。
