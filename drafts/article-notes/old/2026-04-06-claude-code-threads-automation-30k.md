## 要約（3〜5行）

Claude Code を使って Threads（SNS）の投稿を完全自動化し、2週間で3万円のアフィリエイト収益を上げた実践レポート。リサーチャー・アナリスト・ライター・ポスター・フェッチャー・スーパーバイザーの6エージェントが分業して動く。大事故2件（1分で10連投→アカウント凍結、同じ投稿を30回ループ）から「AIの速度をそのまま使うな」「類似度チェックなしでは収束する」という重要な設計教訓を得ている。

## 主なポイント

- **1エージェント＝1タスク の鉄則**：リサーチ・分析・執筆・投稿・データ取得・監視を分離。1人に全部任せると精度が落ちる
- **AIは最適解に収束する**：類似度チェックなし・パターンローテーションなしだと同じ投稿を繰り返す。過去100件との類似度0.85以上で自動棄却が必須
- **人間のリズムに合わせる**：AIは1秒で10投稿生成できるが、最低1時間間隔・1日10タイムスロットに分散しないとBOT判定される

## NOCTAへの関連メモ

SNS の完全自動運用は R-03（自動投稿禁止）・R-10（CEOが最終確認）により NOCTA では適用しない。ただし「AIが同じネタに収束する問題への類似度チェック設計」と「エージェント分業のアーキテクチャ」は音楽制作エージェント設計の参考になる。特に品質スコア自動採点（10項目・7.0未満は棄却）の思想は歌詞・楽曲仕様書の自動品質チェックに応用できる可能性がある。
