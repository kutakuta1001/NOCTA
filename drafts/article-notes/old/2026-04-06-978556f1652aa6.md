## 要約

Codex CLI に実装された Skills 機能の解説記事。Skills は「エージェントが動的に発見・ロードできる指示・スクリプト・リソースのパッケージ」で、Claude Code の slash commands に相当する概念。
YAMLフロントマターの name と description だけを最初に渡し、実際に使う判断がなされたときに本文を読み込む「段階的開示」によりコンテキストを節約できる。
常時必要な内容は AGENTS.md に、状況限定的な知識は Skills に切り分けることがコンテキストエンジニアリングの基本とされている。

## 主なポイント

- Skills の段階的開示（メタデータ先読み→本文遅延ロード）はコンテキスト節約の実践的手法
- グローバルスコープ（~/.codex/skills/）とプロジェクトスコープ（.codex/skills/）の使い分けが可能
- AGENTS.md（常時）vs Skills（状況限定）の二層構造が推奨パターン
- エラー調査→日本語レポート作成を自動化する error-analyzer スキルが具体例として提示

## NOCTAへの関連メモ

NOCTAの~/.claude/commands/（スキル）と~/.claude/agents/（エージェント）の使い分け設計と直接対応。CLAUDE.md に常時必要なルールを、スキルには状況限定的なワークフローを置くという現在の構造が理にかなっていることを再確認できる。
