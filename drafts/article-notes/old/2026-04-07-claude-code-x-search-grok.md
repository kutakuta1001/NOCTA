## 要約（3〜5行）

Claude Code から Grok（xAI API）を「X検索専用マイクロサービス」として呼び出すことで、X のリアルタイムトレンドを Claude Code のワークフローに統合できるという提案。Claude Code 単体では X の最新投稿を取得するのが苦手だが、Grok API 経由で広めのクエリを12個以上投げてタイムラインの空気を抽出するスクリプトを SKILL.md に登録することで解決できる。GitHub に公開スクリプト（x-research-skills）あり。

## 主なポイント

- **問題**: Claude Code や ChatGPT は X のリアルタイム投稿の取得が苦手（情報源の性質の問題）
- **解決策**: Grok（xAI API）を X 検索専用レイヤーとして挟む。エンジニアなら30分で実装可
- **スクリプト設計**: 広めのクエリを12個以上自動生成 → 繰り返し登場する固有名詞を3〜5クラスターに集約 → min_faves/min_retweets オペレータでバズを絞り込む
- **SKILL.md 登録**: Grok 呼び出しスクリプトを Claude Code の Agent Skill として登録することで、トレンド収集→要約→論点抽出が自動化される
- **出力フォーマット**: タイムラインの空気（論点クラスター）＋今日の結論（狙うべき3テーマ）＋素材一覧の3部構成

## NOCTAへの関連メモ

`/phase1-trend` の強化案として有望。現在のトレンド調査は web_search ベースだが、Grok API を追加することで X の音楽トレンド（バズ楽曲・話題の制作手法）をリアルタイムで取得できる可能性がある。xAI API は従量課金（1回 約0.1USD）のため、フェーズ1実行時のみ呼び出す設計が NOCTA のコストポリシーに合う。
