## 要約（3〜5行）
Claude Code に、成果物をユーザーに見せる前に自分でセルフレビューさせるための設定方法を紹介した記事ツイート（bookmark 338・like 138 と反応が高い）。出力品質を上げるために「提示前の自己検証ステップ」を組み込む考え方。

## 主なポイント
- Claude Code に出力前のセルフレビュー（自己検証）を強制する設定が可能
- レビュー観点をあらかじめ定義しておくことで、提示物の品質が安定する
- 高エンゲージメント（ブックマーク多数）で実用性が支持されている

## NOCTAへの関連メモ
NOCTA は既に「起案→批評（Sonnet→Opus）」フローと承認ゲートを持つが、各エージェント出力前のセルフレビュー観点を CLAUDE.md/スキルに明文化すると手戻りがさらに減る可能性。
