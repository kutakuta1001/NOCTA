## 要約（3〜5行）
Claude Code と Codex の両方で毎日使うワークフローtips: タスク開始前にプロジェクト関連情報を整理したcontext fileを作成することが最重要とする実践例。
「context files are everything」という主張で、タスク別の一時コンテキストファイルの作成習慣を推奨している。
CLAUDE.md の常時コンテキストとは別に、タスク固有の情報をその場でまとめる使い方が鍵。

## 主なポイント
- タスク開始前にプロジェクト関連情報をまとめたcontext fileを作成する
- CLAUDE.md（常時コンテキスト）とは別に、タスク別の一時コンテキストファイルを用意する
- 事前の情報整理がAI出力の精度を決定的に左右する

## NOCTAへの関連メモ
NOCTAのcontext.md（プロジェクト常時情報）・handoff.md（継続状態）の使い分けと同じ思想。タスク別の一時メモをdrafts/に作成してからエージェントに渡す習慣を強化する方向で応用できる。
