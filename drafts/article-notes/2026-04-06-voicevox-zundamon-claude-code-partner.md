## 要約（3〜5行）

VOICEVOX（無料テキスト読み上げエンジン）を Claude Code の MCP 経由で連携し、作業開始・完了時にキャラクターの声で報告させた体験談。「了解なのだ！さっそく取りかかるのだ」「できたのだ！」などの音声フィードバックにより、Claude Code が無機質なツールから「相棒」に変わった感覚を得た。ゲーミフィケーション効果で Claude Code を触る回数が体感3倍に増え、自然と使い方も上達したという事例。

## 主なポイント

- **VOICEVOX 連携**：MCP 経由で設定可能。セットアップは GitHub リポジトリ（mcp-simple-voicevox）を Claude Code に渡すだけ
- **待ち時間のストレス解消**：「スクワットでもして待ってるのだ！」のセリフで AI 処理中の待機時間が楽しみに変わる。ドーパミン報酬系の活用
- **複数セッション管理に有効**：複数の Claude Code を並列稼働させている場合、音声通知で完了を検知できる

## NOCTAへの関連メモ

NOCTA では R-08（絵文字禁止・ドキュメントへの装飾制限）があるが、VOICEVOX 連携は UX/体験の問題でありドキュメント品質とは無関係。音楽制作中に複数の Claude Code エージェントを並列実行する場面（/phase2-music の Agent Teams）での完了通知として特に有効。設定コストも低いため試す価値がある。
